# Architecture

## 分层设计

仓库采用“规则优先、schema 驱动、prompt 补充、脚本可验证”的四层结构：

1. `schemas/`
   - 定义输入输出契约
2. `rules/`
   - 定义领域 guardrails 与路由约束
3. `skills/` + `prompts/`
   - 定义技能边界、推理步骤、few-shot 与提示模板
4. `scripts/` + `tests/`
   - 用最小程序验证规则与路由是否符合预期

## 核心流程

1. 解析用户问题
2. 抽取关键字段：年份、省份、学校、分数、位次、选科、竞赛背景
3. 识别问题类型
4. 选路由 skill
5. 运行 risk guard 检查信息充分度
6. 输出结构化 answer
