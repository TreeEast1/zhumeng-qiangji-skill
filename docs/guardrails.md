# Guardrails

## Guardrail 的职责

guardrail 不是简单的“拒答器”，而是用于：

- 识别信息是否充分
- 识别是否存在分数类型混淆
- 阻止无证据的具体数值判断
- 阻止模式误判
- 在高风险场景中强制加入 disclaimer

## 核心 guardrail 场景

- 缺少年份却问“能不能进”
- 缺少省份却问“适合报哪些学校”
- 把“综合成绩”当“高考裸分”
- 说“复交南模式学校都有统一入围线”
- 输出具体录取线但没有证据状态

## 输出策略

- `block`
  - 明显错误或高风险虚构，应拦截
- `downgrade`
  - 允许给保守建议，但必须附带不确定性说明
- `pass_with_warning`
  - 内容大体可用，但需补充警告
