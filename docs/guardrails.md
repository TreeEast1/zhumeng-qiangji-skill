# Guardrails

## Guardrail 的职责

guardrail 不是简单的“拒答器”，而是用于：

- 识别信息是否充分
- 识别是否存在分数类型混淆
- 阻止无证据的具体数值判断
- 阻止模式误判
- 在高风险场景中强制加入 disclaimer

## 核心 guardrail 场景

- 缺少年份却问“能不能进”
- 缺少省份却问“适合报哪些学校”
- 把“综合成绩”当“高考裸分”
- 把“加权入围成绩”当“高考裸分”
- 说“复交南模式学校都有统一入围线”
- 把“报名成功直接参加校测”说成所有学校通用规则
- 把“只考面试”说成“更容易录取”
- 输出具体录取线但没有证据状态
- 忽略体测、校测合格线或特招线等前置条件

## 输出策略

- `block`
  - 明显错误或高风险虚构，应拦截
- `downgrade`
  - 允许给保守建议，但必须附带不确定性说明
- `pass_with_warning`
  - 内容大体可用，但需补充警告

## 来自强基策略材料的补充启发

可参考的二手策略材料显示，强基计划中至少存在以下高混淆点：

- 五种典型入围路径并存，不能一概而论
- 加权入围成绩与高考裸分不可直接横向比较
- “复交南 / 出分前校测”是特定模式，不是通用模式
- 部分院校只考面试，但这不等于录取门槛更低
- 录取前置条件除了综合成绩，还可能包括体测、高考线、校测合格线

这些内容适合进入 guardrail 的“高风险误判列表”，但仍需强调：学校级最终结论必须回到当年官方简章。
