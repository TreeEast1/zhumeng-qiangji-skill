# Data Schema

## 核心对象

### `user_query`

用于承载用户输入与初步抽取字段，面向路由与技能选择。

### `user_profile`

用于承载考生背景和咨询阶段，面向择校与服务分流。

### `answer`

用于承载标准化输出，便于：

- 多 skill 汇总
- 评测
- 后续对接前端或服务系统

### `school` / `policy` / `admissions_rule`

用于定义未来知识层对象结构，不要求第一版完整，但要稳定。

## 设计原则

- 优先可验证字段，不依赖模糊自由文本
- 允许 `null`，但不允许无说明的硬推断
- 用枚举控制高风险字段的可选值
